# ascii-poker

Мультиплеерный Texas Hold'em с ASCII-интерфейсом в терминале. Нативный клиент и авторитетный TCP-сервер на C++20 — без веб-стека и без внешней среды выполнения, кроме C++ toolchain.

Карты, стол и элементы управления рисуются в терминале через [FTXUI](https://github.com/ArthurSonzogni/FTXUI). Вся игровая логика на сервере; клиент отображает состояние и отправляет действия игрока.

**Языки интерфейса:** русский и английский (переключение клавишей `L`).

**English documentation:** [README.md](README.md)

## Возможности

- Вход по логину и паролю; новым игрокам автоматически начисляется стартовый стек
- Баланс фишек в SQLite, синхронизация на границах раздачи (после блайндов и после вручения банка)
- Лобби: создание/вход/выход из комнат, комната ожидания, игра за столом
- Side pots, all-in, стандартные раунды ставок Hold'em
- Интерфейс EN/RU с сохранением языка
- Юнит- и интеграционные тесты (Catch2)

## Быстрый старт

```bash
cmake -B build -DCMAKE_BUILD_TYPE=Debug
cmake --build build
ctest --test-dir build
```

Или скрипт (чистая пересборка):

```bash
./build.sh              # собрать всё
./build.sh poker_client # собрать один target
```

**Системные пакеты** (Ubuntu/Debian): `libsqlite3-dev`, `libssl-dev`

### Запуск

Терминал 1 — сервер (порт по умолчанию `12345`):

```bash
./build/server/poker_server
./build/server/poker_server 8080   # свой порт
```

Терминал 2 — клиент:

```bash
./build/client/poker_client
./build/client/poker_client 127.0.0.1 8080   # свой хост/порт
```

Аргументы CLI переопределяют значения по умолчанию. Клиент читает и пишет `~/.ascii-poker.conf` (логин, хост, порт, язык — пароль не сохраняется).

## Аккаунты и база данных

Сервер хранит аккаунты в **SQLite**. Пароли хешируются **PBKDF2-SHA256** (OpenSSL).

| Параметр | По умолчанию | Переопределение |
|----------|--------------|-----------------|
| Файл БД | `ascii-poker.db` рядом с бинарником `poker_server` | `ASCII_POKER_DB=/path/to/file.db` |

Миграции схемы выполняются при старте сервера (текущая версия схемы: **2**).

### Вход

Клиент отправляет сообщение `login` (логин + пароль). При первом входе создаётся аккаунт с **1000 фишек**. Сервер отвечает `welcome` с текущим балансом.

**Логин:** 1–32 символа, без управляющих символов.

**Пароль:** 4–128 символов, минимум одна буква и одна цифра, без управляющих символов.

### Баланс фишек

- Стеки загружаются из БД при старте игры в комнате.
- Баланс записывается обратно после постинга блайндов и после завершения каждой раздачи (в том числе при выходе игрока посреди руки).
- Для создания или входа в комнату нужно минимум **20 фишек** (big blind).
- Баланс **0** — банкротство: войти в аккаунт можно, оставаться в лобби можно, за стол сесть нельзя, пока снова не наберётся достаточный стек.

## Клиент

### Экраны

| Экран | Содержимое | Основные действия |
|-------|------------|-------------------|
| **Login** | Логин, пароль, хост, порт | `Join`, `Lang`, `Quit` |
| **Lobby** | Список комнат и число игроков | `Refresh`, `Create`, `Join`, `Leave`, `Quit` |
| **Create room** | Название и max игроков | `Confirm`, `Cancel` |
| **Waiting room** | Игроки за столом, метка хоста | Хост: `Start` (≥2 игроков), `Leave`, `Quit` |
| **In game** | Стол, карты, банк, панель действий | `Fold` / `Check` / `Call` / `Raise`, пресеты, `Leave` |
| **Disconnected** | Потеря соединения | `Reconnect`, `Quit` |

Комнаты с пометкой **(in game)** недоступны для входа. **Reconnect** открывает новое TCP-соединение и выполняет вход снова, если пароль ещё в поле логина.

### Горячие клавиши

Работают, когда фокус не в текстовом поле.

| Контекст | Клавиши | Действие |
|----------|---------|----------|
| Lobby | `j` / `k` или `↑` / `↓` | Выбор комнаты |
| Lobby | кнопки `▲` / `▼` | То же |
| In game (ваш ход) | `f` | Fold |
| In game (ваш ход) | `c` | Check или Call |
| In game (ваш ход) | `r` | Фокус на сумму рейза |
| Любой экран | `[` / `]` | Прокрутка лога |
| Любой экран | `L` | English ↔ Русский |
| Login | `Enter` | Join (как кнопка `Join`) |

На своём ходе доступны пресеты рейза: **Min**, **½ pot**, **Pot**, **All-in**. Кнопка Call показывает сумму, если нужен колл (например, `Call 20`).

### Отображение стола

- Тёмная slate-панель; приглушённые подсветки фокуса
- Красные масти (♥ ♦): coral; чёрные (♣ ♠): светло-серые
- Активный игрок: префикс `>>` золотым
- Дилер / блайнды: `[D]`, `[SB]`, `[BB]`
- Ваше место: суффикс `(you)`
- Fold / all-in: `[fold]`, `[all-in]`
- Подсказка по стартовой руке и по собранной комбинации после борда
- Баннер шоудауна: победители, размер банка, категория руки

### Файл настроек

`~/.ascii-poker.conf`:

```ini
player_name=Alice
host=127.0.0.1
port=12345
language=ru
```

`language=en` — английский. Настройки сохраняются после успешного входа и при смене языка. Пароль хранится только в памяти на время сессии.

## Правила игры (сервер)

| Параметр | Значение |
|----------|----------|
| Стартовый стек (новый аккаунт) | 1000 фишек |
| Small blind | 10 |
| Big blind | 20 |

- Блайнды постятся в начале каждой раздачи; кнопка дилера смещается.
- Ставки идут в стандартном порядке Hold'em; при нескольких all-in считаются side pots.
- Хост может нажать **Start** при ≥2 игроках; игра также стартует автоматически, когда комната заполнена.
- Игрок, вышедший или отключившийся посреди руки, автоматически фолдит. Сессия продолжается, если в комнате остаётся ≥2 игроков; если у менее двух игроков есть фишки — сессия завершается.
- Список комнат рассылается всем клиентам при создании, входе, выходе и старте игры.

## Протокол

TCP, асинхронный I/O (Asio). Сообщения — JSON с префиксом длины: 4 байта big-endian + UTF-8 тело.

```json
{ "type": "login", "data": { "username": "Alice", "password": "secret1" } }
```

Справочники:

- Типы сообщений: `common/include/poker/protocol.h`
- Коды ошибок: `common/include/poker/error_codes.h`
- Валидация auth (общая для клиента и сервера): `common/include/poker/auth.h`

## Структура проекта

```
common/       Общая логика покера, протокол, auth, framing (poker_core)
server/       TCP-сервер, лобби, комнаты, сессии, SQLite (poker::server)
client/       Терминальный клиент FTXUI (poker::client)
third_party/  Vendored: asio, spdlog, nlohmann/json, FTXUI, Catch2
tests/        Юнит- и интеграционные тесты
scripts/      check-format.sh
```

Результаты сборки:

- `build/server/poker_server`
- `build/client/poker_client`
- `build/tests/poker_tests`

Файлы SQLite (`*.db`, `*.db-wal`, `*.db-shm`) в `.gitignore` и по умолчанию создаются рядом с бинарником сервера.

## Разработка

Тесты:

```bash
ctest --test-dir build --output-on-failure
```

Проверка форматирования:

```bash
bash scripts/check-format.sh
```

Применить clang-format:

```bash
git ls-files '*.cpp' '*.h' | grep -v '^third_party/' | grep -v '^tests/external/' | grep -v '/ftxui/' | xargs clang-format -i
```

CI (GitHub Actions) собирает проект и запускает тесты при каждом push.
